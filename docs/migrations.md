# Migrations

## Bug fixes in `001_initial.sql`

Four bugs were present in the hand-written initial migration. All fixed before running.

---

### Bug 1 — `cast` is a reserved keyword

`cast TEXT[]` causes an immediate parse error. `CAST(x AS INT)` is built-in SQL syntax — PostgreSQL can't disambiguate a column named `cast`.

**Fix:** renamed to `cast_members TEXT[]`.

---

### Bug 2 — Index on a non-existent column

`CREATE INDEX idx_content_type ON streaming_content(content_type)` — `content_type` was never defined as a column above it.

**Fix:** added `content_type VARCHAR CHECK (content_type IN ('movie', 'series'))` to the table. Filtering by type is a core feature; the index is correct.

---

### Bug 3 — Foreign key to a table that doesn't exist

`watch_history.user_id REFERENCES users(id)` — no `CREATE TABLE users` existed anywhere in the file. PostgreSQL enforces FK targets at creation time.

**Fix:** added a `users` table before `watch_history` with `id`, `email`, `password_hash`, `created_at`.

---

### Bug 4 — Trigger cascade failure from Bug 1

The `set_updated_at` trigger on `streaming_content` failed with `relation "streaming_content" does not exist` because Bug 1 prevented the table from being created at all.

**Fix:** resolved automatically once Bug 1 was fixed.

---

### Why the order matters

PostgreSQL executes DDL sequentially — a statement failure doesn't roll back the file, it just skips that statement. Bug 1 failing meant the table never existed, so the index, trigger, and any FK referencing it all failed too. Four bugs, more than four errors. This is why DDL migrations should be wrapped in a single transaction.

## TypeORM generated migrations

All subsequent migrations are generated by the TypeORM CLI — never written by hand.

```bash
pnpm build && pnpm migration:generate --name=DescriptiveName
# review the generated file
pnpm build && pnpm migration:run
```

Extended the `content_type` CHECK constraint to include `'live'`. Also reconciled FK constraint names from the hand-written convention to TypeORM's hashed convention (expected one-time delta).
